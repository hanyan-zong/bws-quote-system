## 已知避坑（继承自母库 CLAUDE.md）

### docx-js 中文引号
- 所有中文对话引号必须用 `「」`，不用 ASCII 单引号；否则 JS 字符串语法被破坏
- 整段中文用反引号 `` ` `` 包裹更安全

### SVG 中 `&` 必须转义
- LibreOffice 渲染 SVG 时遇到未转义的 `&` 直接报 "source file could not be loaded"
- 统一过滤：`re.sub(r'&(?!amp;|lt;|gt;|quot;|apos;|#)', '&amp;', svg_text)`

### CJK 字体
- 系统只有 `DroidSansFallbackFull.ttf` 一种 CJK 字体
- DOCX/PPTX 用 "Microsoft YaHei" 名称会自动 fallback
- PDF 走 WeasyPrint 是中英混排唯一可靠方案

### openpyxl 读公式
- LibreOffice 转过的 xlsx 才有缓存值；先 `libreoffice --headless --convert-to xlsx` 再用 openpyxl 读

### Cowork 沙箱权限
- `/sessions/.../mnt/balijob/` 可写
- 所有交付物用 `/tmp/` 中转再 `cp` 到目标目录

### SQLite on mounted FUSE filesystem
- `/sessions/.../mnt/balijob/` 是 FUSE 挂载, 不支持 SQLite 的文件锁定
- 直接在该目录建 .db 会报 `disk I/O error`
- **解决方案**: 启动时设置 `BWS_DATABASE_URL="sqlite:////tmp/bws_quote.db"` 或在 config.py 默认改用 /tmp
- 或者切到 PostgreSQL — v0.2 推荐
- start.sh 应该自动检测并切换 DB 路径

### Pydantic v2 + Python 3.10 字段名遮蔽类型
- 字段命名 `date: date | None = None` 会导致 `unsupported operand type(s) for |: 'NoneType'`
  - 因为字段名 `date` 与 `from datetime import date` 同名,Pydantic 评估注解时 `date` 解析为 None
- **解决方案**: schemas.py 中 `from datetime import date as _date`, 用 `Optional[_date]` 而非 `date | None`
- 或升级 Python 到 3.11+ (推迟评估行为不同)

### pip 安装
- 必须加 `--break-system-packages`
- 对于 weasyprint/pdfplumber 等需要系统库的，先 apt-get 安装依赖

## 设计原则不可让步

1. **AI 优先于规则**：能让模型判断的不写硬规则
2. **前端零构建**：管理者也能直接改 HTML 文件
3. **成本审计**：每个报价能反向查到由哪些资源/规则/AI 输出生成
4. **失败可读**：任何错误必须给出"用户能看懂下一步该做什么"的提示，不能只丢 stack trace
5. **风险护栏**：赌自费有硬上限，行程合理性有硬约束，AI 不能凌驾业务规则
