### `distances` 距离矩阵 ★ 核心新表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | |
| from_type | VARCHAR(20) | hotel / attraction / restaurant / airport |
| from_id | INTEGER | 源 POI ID |
| to_type | VARCHAR(20) | 同上 |
| to_id | INTEGER | 目的 POI ID |

> **设计要点**：双向存储（A→B 和 B→A 各一行）以便简单查询；初始化用脚本批量生成；后续可对接 Google Distance Matrix API 自动更新。

## 索引策略

- `attractions(destination_id, area)` 复合索引：按区域筛景点
- `distances(from_type, from_id, to_type, to_id)` 唯一索引：避免重复
- `hotel_rooms(hotel_id, valid_from, valid_to)` 复合索引：按日期查可用房型
- `quotes(start_date, status)` 复合索引：按状态+日期检索
