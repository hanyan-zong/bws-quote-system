## v0.2 增量：车型-距离绑定校验

新增"车型选了不合适会走不动"的硬约束。

### 数据层增量

**Vehicle 新增字段：**
- `max_single_leg_minutes`：该车型单段最长可驾驶分钟数（35 座大车山路单段不能 ≥2h）
- `max_daily_minutes`：该车型全天最长驾驶（覆盖全局 `time_budget_config`）
- `terrain_note`：地形限制说明（前端可展示给客户）

**Distance 新增字段：**
- `vehicle_max_seat`：该路段允许的最大座位数（>此值直接 fail）
- `vehicle_warn_seat`：该路段推荐最大座位数（>此值 warn 但允许）

### 校验流程增量

每段距离校验时，除原有的总驾驶/车型禁入区域外，新增：
1. 若 `vehicle.seat_count > distance.vehicle_max_seat` → **error: 该路段不能用此车型**
2. 若 `vehicle.seat_count > distance.vehicle_warn_seat` → warning
3. 若 `leg_minutes > vehicle.max_single_leg_minutes` → **error: 单段超时**
4. 若 `total_drive > vehicle.max_daily_minutes`（车型自身的限制优先于全局）→ **error**

## 一、问题定义

B 端旅行社的销售或客户在选行程时，常出现：
1. 一天里把跨岛南北的景点排在一起（金巴兰晚餐 + 京打玛尼火山日出 — 单程 2 小时，根本来不及）
2. 酒店选在乌布，但当天行程是塔巴南海神庙 + 努沙杜瓦 — 早上从乌布出发就要 2 小时
3. 17 座车带去 Monkey Forest 街（车型限行）
4. 节假日还按平日时间排，结果堵车 2 小时
5. 自由活动天酒店在偏远地区，客人哪里都不方便去

**校验目标**：在用户点击"生成报价"前，把这些不合理项**全部拦截 + 给出可行替代方案**。

## 二、校验维度（按优先级）

### Tier 1 · 硬约束（必须 fail，不能放过）

| # | 规则 | 实现 |
|---|------|------|
| 1 | 单日总驾驶时间 > 上限（默认 5h） | 路径求和 + 堵车系数 |
| 2 | 车型不能去某些区域（35 座禁入清单） | 检查 vehicles.restrictions vs attractions.area |
| 3 | 景点闭馆日 / 闭馆时段 | open_time/close_time + 当日抵达时间比对 |
| 4 | 自费/SPA 最低人数不达标 | min_pax 检查 |
| 5 | 抵离日酒店离机场超过 1.5 小时 | airport_buffer_minutes |

### Tier 2 · 软约束（warn 但允许）

| # | 规则 | 警告 |
|---|------|------|
| 1 | 单日驾驶 4h-5h 之间 | "节奏紧凑，建议优化顺序" |
| 2 | 早上首站距离酒店 > 90min | "建议改为下午景点" |
| 3 | 同一区域连续两天重复 | "Day 2 和 Day 3 都在乌布，建议拉开" |
| 4 | 餐厅与上一站距离 > 30min | "晚餐绕路，建议换 X 餐厅" |

### Tier 3 · AI 评估（自然语言洞察）

把当日完整路线交给 Claude，让它输出一段评估：
```
"Day 2 路线评分 6/10。圣猴森林→德格拉朗梯田→京打玛尼是合理的爬升路线，
但下午回乌布吃晚餐再去金巴兰看夕阳意味着要再往西南开 1 小时，
晚上 9 点才能到酒店。建议把金巴兰夕阳挪到 Day 3 抵离日返程顺路。"
```

## 三、距离矩阵的来源与维护

### 3.1 数据来源（按优先级）
1. **管理员人工录入**（最准，但工作量大）
2. **导游回报**（每团结束让导游填实际车程）
3. **Google Distance Matrix API**（自动批量回填，付费但便宜）
4. **OpenStreetMap + OSRM**（免费但精度略差）

### 3.2 字段语义
- `normal_minutes`：周二到周四 10:00-15:00 的车程
- `peak_minutes`：早 7-9 点 / 晚 17-19 点
- `holiday_minutes`：周末 / 节假日 / 春节

### 3.3 缺数据时的兜底
- 同 area 内默认 15 min
- 跨 area 默认 30 min  
- 跨 destination 默认 ∞（不允许同日）
- 缺数据自动打 warning，提醒管理员补录

## 四、一日游模板的运作机制

### 4.1 模板举例

**模板：乌布文化一日游**
- 关联景点（顺序）：圣猴森林（90min）→ 德格拉朗梯田（60min）→ 乌布皇宫（45min）→ 乌布市场（60min）
- 关联餐厅：午餐 - Bebek Bengil（鸭饭）/ Murni's Warung
- 推荐用车：7-17 座
- 总时长估算：8 小时（含车程）
- 难度：easy

**模板：京打玛尼火山日出团**
- 关联景点：京打玛尼日出点（120min，凌晨 4 点出发）→ 早餐 → 圣泉寺（60min）→ 德格拉朗梯田（60min）
- 关联餐厅：早餐山顶 Café 1 / 午餐山区
- 推荐用车：7 座（山路不便大车）
- 总时长估算：10 小时
- 难度：intense（凌晨出发）

### 4.2 用户选模板的两种模式

**模式 A · 严格模板模式（推荐新手客户）**
- 客户选完模板，景点/餐厅下拉框里**只显示该模板内的选项**
- 完全保证可行性，零失败率

**模式 B · 自由组合模式（适合资深销售）**
- 客户可任意搭配跨模板的景点
- 但 Feasibility Engine 会用全部规则严格校验
- AI 可能给出"建议拆成 Day 2 + Day 3"的 suggestion

### 4.3 模板由谁维护

- 管理员在 `/admin/templates` 页面维护
- AI 助手能根据"过去 100 单成功行程"自动归纳模板（v0.2 功能）
- 模板有版本号，老报价仍能查到当时模板

## 五、校验算法伪代码

```python
def check_feasibility(quote_id: int) -> FeasibilityReport:
    quote = load_quote(quote_id)
    config = load_time_budget_config(quote.destination_codes)
    days_report = []
    overall_feasible = True
    
    for day in quote.days:
        report = DayReport(day_index=day.day_index)
        
        # 1. 收集当日所有 POI（按时间顺序）
        pois = build_day_route(day)  # 酒店→景点1→景点2→餐厅→景点3→酒店
        
        # 2. 计算总驾驶时间（带堵车系数）
        total_drive = 0
        for i in range(len(pois) - 1):
            d = get_distance(pois[i], pois[i+1])
            coef = pick_coef(pois[i], pois[i+1], day.date, config)
            total_drive += d.normal_minutes * coef
        report.drive_minutes = total_drive
        
        # 3. Tier 1 硬约束
        if total_drive > config.max_drive_minutes_per_day:
            report.errors.append(f"总驾驶 {total_drive}min 超过上限 {config.max_drive_minutes_per_day}min")
            overall_feasible = False
        
        if day.vehicle and day.vehicle.restrictions:
            for poi in pois:
                if any(r in poi.area for r in day.vehicle.restrictions):
                    report.errors.append(f"{day.vehicle.vehicle_type} 不能进入 {poi.name}")
                    overall_feasible = False
        
        # 4. Tier 2 软约束
        if total_drive > config.max_drive_warn_minutes:
            report.warnings.append(f"驾驶时间 {total_drive}min 偏紧")
        
        first_attraction_drive = compute_drive(day.hotel, pois[1])
        if first_attraction_drive > config.hotel_to_first_max_minutes:
            report.warnings.append(f"早上首站距酒店 {first_attraction_drive}min")
        
        # 5. Tier 3 AI 评估（异步，不阻塞）
        report.ai_review_pending = True
        schedule_ai_review(quote_id, day.day_index, pois)
        
        # 6. 给出替代方案（基于规则 + AI）
        if report.errors:
            report.suggestions = generate_suggestions(day, pois, report.errors)
        
        days_report.append(report)
    
    return FeasibilityReport(
        overall_feasible=overall_feasible,
        days=days_report
    )
```

## 十、与赌自费的联动

如果某天行程被改为"自由活动"，Feasibility Engine 不再校验该天的路线，但会：
- 提醒"该天酒店附近的可推荐自费项目"
- 把该天加入 `free_days` 计数 → 触发 Gamble Engine
