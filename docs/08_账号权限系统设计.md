## 一、决策树（5 题已定）

| 题 | 答案 | 影响 |
|---|------|------|

## 四、邀请-注册流程

### A · 邀请 super_admin → agency_owner

```
1. super_admin 在系统里:
     POST /api/v1/agencies  → 创建 agency
     POST /api/v1/invitations  body={agency_id, role: "agency_owner", expires_in_days: 7}
     → 拿到 code "bws-XXXXX"

2. super_admin 发邀请链接给 B 端老板:
     https://yourdomain.com/register?code=bws-XXXXX

3. B 端老板访问页面:
     GET /api/v1/auth/invitation/bws-XXXXX
     → 页面显示 "您被 XX 邀请加入 上海康辉，请设置账号"

4. B 端老板填表:
     POST /api/v1/auth/register
     body={code, username, password, display_name, phone}
     → 后端校验 code → 创建 user (role=agency_owner) → 自动登录
```

### B · agency_owner → agent

owner 进系统后在 "我的团队" 面板生成下级邀请码：
```
POST /api/v1/invitations
body={agency_id: <自动取自己的>, role: "agent", expires_in_days: 30, note: "李业务员"}
```
（owner 无法生成 super_admin / 跨 agency 的邀请，后端校验拒绝）

## 五、密码 & 会话

### 密码策略

| 项 | 规则 |
|---|------|
| 最短长度 | 8 字符 |
| 复杂度 | 至少含字母 + 数字（不强制特殊字符，简单可用即可） |
| 哈希 | bcrypt cost=12 |
| 重试 | 5 次失败 → 该 username 锁定 15 分钟 |
| 首登强制改 | super_admin 创建/重置后 `force_password_change=True` |

### 会话机制

继承 v0.2.1 的 cookie 路线，**升级为带 user_id**：

```
cookie = "{user_id}|{expires_unix}|{HMAC-SHA256(user_id|expires, BWS_AUTH_SECRET)}"
```

- 优点：无依赖（不用 PyJWT），实现简单，已有代码改动小
- 缺点：撤销前要等 cookie 过期；可接受（B 端场景不像金融需要立即吊销）
- 续签：每次请求若剩余有效期 < 1 天 → 生成新 cookie 续签

未来需要立即吊销时 → 加 `users.session_version` 字段，cookie 带版本号校验。

## 六、字段过滤策略（隐藏成本核心）

### 后端层

新增 `app/utils/permissions.py`：

```python
def filter_quote_fields(quote_dict: dict, user: User | None) -> dict:
    """根据用户角色裁剪 quote 输出字段."""
    if user is None or user.role in ("super_admin", "agency_owner"):
        return quote_dict  # 完整可见
    # agent / viewer:
    sensitive = ["cost_idr_total", "cost_cny_total",
                 "profit_cny_per_pax", "gamble_cny_per_pax"]
    return {k: v for k, v in quote_dict.items() if k not in sensitive}


def filter_resource_fields(rows: list[dict], rtype: str, user: User | None) -> list[dict]:
    """资源库返回 — agent/viewer 看不到 cost_idr_*"""
    if user is None or user.role in ("super_admin", "agency_owner"):
        return rows
    cost_keys = {
        "hotel_room": ["cost_idr_low", "cost_idr_high"],
        "attraction": ["ticket_idr_adult", "ticket_idr_child"],
        "restaurant": ["cost_idr_per_person"],
        "vehicle": ["cost_idr_per_day"],
        "guide": ["cost_idr_per_day"],
        "spa": ["cost_idr_per_person"],
        "water_activity": ["cost_idr_per_person"],
        "afternoon_tea": ["cost_idr_per_person"],
        "optional_tour": ["cost_idr"],
    }
    keys = cost_keys.get(rtype, [])
    return [{k: v for k, v in row.items() if k not in keys} for row in rows]
```

每个 resource list endpoint 在 return 前过一道 `filter_resource_fields`。

### 前端补救（不显示空字段）

`components.js` 的 ResourceManager 表格：用 `v-if` 检测字段存在，存在才显示该列。

### 价位档位（agent 看的替代品）

v0.2.5 已有 "价格档位" 字段（hotel: 经济/中端/高端/奢华），属性 `tier`（不是数字）。这条字段对 agent 可见，给业务员一个"贵不贵"的直觉，不暴露具体 IDR。

## 八、迁移计划（升级影响最小）

### 现有用户的转换

v0.3 是单租户口令门，`.env` 里 `BWS_AUTH_USERNAME` / `BWS_AUTH_PASSWORD`。
v0.4 启动时 `init_db()` 自动检查：
- 如果 `users` 表为空 → 用 `.env` 的账号自动建一个 super_admin 用户
- 默认建一家 agency `name="本社"` 关联，allowing 现有报价 owned by super_admin 不丢

### 旧 cookie

旧 cookie 没有 user_id，新 middleware 检测到旧格式 → 强制重新登录。一次性迁移成本可接受。

### 现有 quote.created_by 字段

之前是字符串（"admin"），保留兼容。新增 `created_by_user_id` int 字段，新建报价填这个，老的 fallback 到字符串。
