## 立刻要做(有 1-2h 就能干)

### P0-1 Alembic 迁移工具 [1.5h]
**痛点**: `database.py` 的 `additive_columns` 已加到 10 行 (v0.4 加了 6 列 + v0.5 加了 4 列). 再加几次就乱了, 也没法回滚.

**做什么**:
```bash
docker compose exec bws-quote bash
cd /app/backend
pip install alembic
alembic init alembic
# 编辑 alembic/env.py:
#   from app.database import Base
#   target_metadata = Base.metadata
#   from app import models  # noqa - 注册全部
alembic revision --autogenerate -m "v0.5 baseline"
alembic upgrade head
```

之后:
- 新模型字段 → `alembic revision --autogenerate -m "..."`
- `Dockerfile.entrypoint.sh` 改成: `alembic upgrade head` 替代 `init_db.py`
- 删掉 `database.py` 里的 `additive_columns` 整段

**风险**: 旧库已存在 → autogenerate 可能漏检测 (因为 ALTER TABLE 加的列没在 alembic 历史里). 解决: 第一个 revision 要手写 `op.execute("CREATE TABLE IF NOT EXISTS ...")` 跳过已建表, 或者 `alembic stamp head` 假定起点.
