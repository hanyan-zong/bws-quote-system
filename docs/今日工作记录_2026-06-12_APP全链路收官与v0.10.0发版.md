### 4. 报价单导出 + 安全修复
- **fix(security)** mobile `e12f97b`: 模板 `getEnvBaseUrl` 在 mp-weixin 任何 envVersion 下把 baseUrl 覆盖成模板作者服务器 `https://ukw0y1.laf.run` — **登录请求含账号密码会外发**。彻底删除, 同步上游模板时严禁带回 (SKILL 踩坑⑧)。
- **导出**: interceptor 补拦 `downloadFile` (自动拼 baseUrl+Bearer; **401 无感刷新不覆盖此通道**, 调用前先 `tryGetValidToken`); H5 走 `a[download]`, mp/APP 走 `uni.openDocument({showMenu})` 微信内直接转发; 管理层角色文件含成本 (后端 can_see_costs 语义) → 下载后 toast 提醒。

### 5. 修 web PDF 导出 (从 v0.5 起就是坏的) + 两个测试暴露的真问题
- **PDF 500 根因**: weasyprint 缺 GTK3 原生库; **cffi 的 `ffi.dlopen` 只认 PATH, 不吃 `os.add_dll_directory`** (ctypes 测通 ≠ weasyprint 能跑)。装 GTK3-Runtime 3.24.31 (静默 `/S`) + bin 写入用户 PATH; 又踩 weasyprint 62.3 配 pydyf 0.12 炸 `'super' object has no attribute 'transform'` → 一起升 weasyprint 69。实测三格式 200, PDF 两页中文完美。
- **真 bug — 翻页重复/丢单** (`fab6dbd`): 列表 `order_by(created_at desc)` 同秒多单顺序未定义; flaky 的 pagination 测试暴露。加 `id desc` tiebreaker 根治。
- **seasonal e2e flaky 根治** (同 commit): 根因 Windows 偶发 **WinError 10055** (socket 缓冲区耗尽, 混跑 subprocess CLI 测试时) 打断 teardown → 残留 surcharge 让后续计价翻倍连锁失败 (症状: 单跑全过/混跑随机挂)。fixture 改**自愈模式**: setup 先按 PREFIX 清残留 + teardown best-effort, 任何中断不传染下一次。
- 过程坑: `bws server stop` 有交互式 `[y/N]` 确认, 后台跑会卡死; uvicorn 是 multiprocessing 父子树, 只杀父进程会留孤儿 worker 继续占 8000。

### 6. 导出文档页脚版本动态化 (`a4a9b2b`)
- PDF 模板 / docx_builder / context.meta.system 三处客户可见的 "v0.5" 写死字符串, 统一动态取 `app.__version__`, bump 后自动跟上。

### 7. P0 settings 重构 (`29b48d7`, 挂了近一个月)
- `config.Settings` 的 database_url/auth_* 改 **property** (每次读 env), 其余字段 YAGNI 不动。
- `database.engine/SessionLocal` 改 `get_engine/get_sessionmaker` (lru_cache 延迟创建), 旧名经 **PEP 562 模块级 `__getattr__`** 兼容 — 全部函数内 import 的调用方零改动。
- `tests/web/conftest.py` 删除 sys.modules reload hack; **零新依赖** (没引 pydantic-settings)。
- 154 PASS 两轮 + `bws db stats` 真实环境冒烟。
