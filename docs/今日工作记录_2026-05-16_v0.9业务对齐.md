## 五、踩坑:CLI 测试 collect 时污染父进程 settings

### 症状

混跑时 web 测试 8 个 ERROR `admin login failed: 403`,单跑 web 测试 14/14 PASS。

### 排查过程

1. 加诊断 fixture 打印 `engine.url` + admin 状态
2. 看到 `engine.url=sqlite:///backend/data/bws_quote.db`(**主 DB!**)而不是 mkdtemp tmp
3. admin `locked_until=2026-05-16 12:25:14` — 被锁定(我之前手动浏览器登录尝试过)

### 根因

- `test_cli_data_import_json.py` 顶部 `from app.cli.data_cmd import _parse_import_summary`
- pytest collect 阶段就触发 `app.cli.data_cmd` import → `from ..config import settings`
- `app.config` module-level: `settings = Settings()` → class body `database_url: str = os.getenv(...)` 在 import time 求值
- 此时 web/conftest.py **还没运行**,`BWS_DATABASE_URL` 还没 set → `settings.database_url` 锁定成主 DB
- 后续 web/conftest.py 即使 set env 也晚了 — `from app.database import engine` 时 engine 用已锁定的 settings.database_url

### 修复(防御性 sys.modules reload)

```python
# web/conftest.py 顶部
os.environ["BWS_DATABASE_URL"] = f"sqlite:///{_TMP_DB_PATH}"
# 防御: CLI 测试可能已经 import 了 app.config 锁定主 DB. 清掉强制 reload
for _mod in list(sys.modules):
    if _mod == "app" or _mod.startswith("app."):
        del sys.modules[_mod]
from app.database import engine, init_db  # 重新 import, settings 拿 tmp DB

assert str(engine.url).endswith("test.db"), f"engine.url={engine.url} — reload 没生效"
```

加 sanity assert 防御未来类似坑。

### 深层教训

`settings = Settings()` 这种 **class-level os.getenv** 是 BWS v0.x 简化设计,在 import time 锁定配置 → 不利于测试隔离。**真正干净的修法**:settings 改 pydantic-settings + FastAPI dependency override。但那是另一个重构 — 见下次方向 v2。
