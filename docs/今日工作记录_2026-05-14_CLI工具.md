## 四、踩到的坑

### 1. `bws.bat` UTF-8 编码 → cmd 当 GBK 解析,全部乱码

**症状**:
```
'��到' is not recognized as an internal or external command
'8"' is not recognized as an internal or external command
```

**根因**:Write 工具默认写 UTF-8,但 Windows .bat 在中文系统下 cmd 用 cp936 解析。

**修复**:.bat 全文改纯 ASCII (注释/echo 全英文),用户消息允许丢一点中文友好度换稳定性。

> 旧记忆 `[Windows + Python 环境踩坑]` 已经说过类似问题,但只针对 .py 脚本。本次确认 **.bat 同样需要纯 ASCII**,已沉淀到本文档 + memory。

### 2. setuptools 找不到 `app` 包

backend/app 的非常规布局,需要在 `pyproject.toml` 里:
```toml
[tool.setuptools]
package-dir = {"" = "backend"}

[tool.setuptools.packages.find]
where = ["backend"]
include = ["app*"]
```
否则 `pip install -e .` 会装空包,`bws.exe` 一调就 ImportError。

### 3. `db query` 写 SQL 兜底

argparse 不能阻止用户传 `bws db query "DROP TABLE quotes"`,所以入口里 strip+lower 后 startswith 检查 `insert|update|delete|drop|alter|create|truncate|replace`,命中即拒。

不依赖 SQLAlchemy 的 readonly 模式 — 不想引入额外配置。这种字符串前缀检查对 CLI 这种"开发者自用"够用,真要绕过自己连 sqlite3。
