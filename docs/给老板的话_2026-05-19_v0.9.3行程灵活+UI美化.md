## 这次到底做了什么

### 1. 每天可以选"行程时长" — 全天 / 半天 / 抵达日 / 送机日
每天卡片头部多了一组按钮,选项:
- **全天** — 老行为,车导算 1 整天
- **半天** — 半天用车,车导按 0.5 天算费
- **抵达日** — 客人下午才到,只玩半天,车导按 0.5 天
- **送机日** — 早上送机就走;**自动不算住宿**(前一晚酒店已含);**早餐自动打勾**;车导按 0.5 天

最后一天选"送机日"之后,酒店字段会消失,黄色提示"🛫 送机日 — 不入住酒店,早餐由前一晚酒店提供;车导按半天计费"。

### 2. 一键"复制此天到下一天"按钮
每天卡片头部多了绿色 **📋 复制** 按钮。点击后:
- 复制全部:酒店 / 房型 / 用车 / 导游 / 含早 / 餐厅 / 下午茶 / SPA / 水上 / 景点 / 备注
- 在它后面插入一天,后续天的编号自动顺延
- 日期不复制(你自己填新日期)

适合连续两天同酒店、同司机、相似景点的情况,以前要逐项重选。

### 3. 景点列表变好看 — 路线图样式
之前:`[#1] 圣猴森林 (乌布)` 平铺
现在:
- 第 1 个景点 🟢绿圆 + "▶ 出发"标签
- 中间景点 🔵蓝圆
- 最后一个 🔴红圆 + "■ 终点"标签
- 序号圆之间用竖灰线连接,做出"路线"感
- 景点名加粗,区域单独显示 `📍 乌布`
- 删除从"删除"文字按钮改成红色 `×`

### 4. 长者输入框修好(之前点不进去填数字)
原因:输入框被挤太窄,数字框宽度<20px 根本看不见。
修复:输入框宽度撑满列;同时把"老年(55+)"改成"长者"(跟"成人/儿童"对齐)。
顺手:目的地下拉也撑满列,以前会被挤压遮挡。

## 为什么要做

- **行程类型** 是你直接提的需求 — "最后一天不含住宿只含早餐,半天用车走半天行程然后送机"。这套类型完美对应你说的场景,以后填末日不用再手动清空酒店字段。
- **复制天** 减少重复劳动 — 一单 5 天可能有 3 天住同一家酒店、用同一辆车,以前每天都要重新选 10 个字段。现在 1 个按钮搞定。
- **景点路线美化** 你昨天反馈 "显示不清晰、复杂、混乱,要让客户看得懂"。重新设计成路线图样式。
- **长者填空 + 目的地遮挡** 你上一轮反馈的 bug。

## 风险 (实事求是)

- **已知小问题**:无。67 个自动检查全过(原 59 + 新 8 个 day_type 测试)。
- **要观察一段时间的**:
  - 半天用车系数固定 **0.5**。如果你的实际场景半天车费不是 0.5,需要调整,告诉我(下次给 vehicle 加 `cost_idr_per_half_day` 字段独立定价)。
  - 送机日**没有"额外送机费"概念**。如果实际上送机要加费用(过路费/排队费),目前 0.5 倍用车没算这个,需要时告诉我。
- **明确没做的**:
  - **抵达日没有自动逻辑**(不像送机日强制去酒店)。抵达日的酒店是要入住的(客人晚上要睡),所以酒店保留可填。如果你想让抵达日也自动有什么默认行为,告诉我。
  - **没改一日游模板编辑器**(那个只有 admin 用,客户看不到,先不优化)。
  - **复制天不复制日期**(顺延一天的语义模糊,可能跨月、跨季节;让你自填更安全)。

5. 加 3 个景点 → 应该看到 🟢1→🔵2→🔴3 圆形徽章 + 出发/终点标签 + 灰色连接线
6. 在"基本信息"区,**长者**输入框应该能正常填两位数,旁边目的地下拉不被挡
7. 页脚显示 **v0.9.3**

报价试算:
- 加一辆车(用车成本 1,000,000 IDR/天)
- Day 1 选"全天" → 车费应该 1,000,000
- Day 2 选"半天"或"送机日" → 车费应该 500,000(0.5 倍)
- 详情明细写"用车 商务车 (半天): 500000 IDR" 或 "用车 商务车 (送机日): 500000 IDR"

如果出现"复制按钮不出现 / 送机日酒店没消失 / 景点没圆圈"任何一项,99% 是浏览器缓存,再 Ctrl+F5。
