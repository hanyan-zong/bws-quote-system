## 模块边界

### 1. Resources（资源库）
统一管理各类成本资源：
- **HotelRoom**（酒店房型）：每晚成本（IDR）、淡旺季倍率、最少入住数
- **Restaurant**（餐厅）：人均成本（IDR）、是否含饮料
- **Attraction**（景点）：门票（IDR）、儿童价、是否含导览
- **AfternoonTea**（下午茶）：人均成本
- **SpaPackage**（SPA 套餐）：人均成本、时长
- **WaterActivity**（水上项目）：人均成本（如浮潜、滑翔伞、海钓）
- **Vehicle**（车辆）：日租成本（IDR）、座位数、是否含司机汽油过路费
- **Guide**（导游）：日费（IDR）、语言（中/英/印尼）、级别
- **OptionalTour**（自费项目）：售价（CNY）、成本（IDR）、历史购买率

每个资源支持：标签、目的地、有效期、备注。

### 2. Quotes（报价单）
- **Quote**：报价单主表（客户名、人数、日期、总价、利润、状态）
- **QuoteDay**：每日明细（第几天、是否自由活动、用车、导游、午餐、晚餐）
- **QuoteItem**：每天的具体项目（关联 Resources 表）
- **QuoteSnapshot**：报价快照（用于历史对比与审计）

### 3. AI Parser（AI 文档解析）
- 接收上传文件 → 识别类型（PDF/DOCX/Excel/Image）
- 第一步：本地预解析（pdfplumber/openpyxl/python-docx 抽出结构化文本）
- 第二步：调用 Claude API 做"语义校对+字段归一+缺失补全"
- 第三步：返回结构化 JSON，前端预览后用户确认入库
- 修正记录写入 `logs/ai_corrections.jsonl`（用于持续优化 prompt）

### 4. Gamble（赌自费算法）
- **输入**：行程结构（总天数、自由天数、人数、季节、客户类型）
- **输出**：推荐"敢低于成本"的金额范围 + 中位数 + AI 信心分
- **算法**：见 `04_赌自费算法.md`
- 支持手动覆盖 + 写入实际成单数据反哺模型

### 5. Feasibility Engine（行程合理性校验）
解决"客户不熟巴厘地理，自己拼出来的行程根本走不完"的痛点。

**输入**：QuoteDay 列表（每天的酒店、景点、餐厅、用车）
**输出**：每天的可行性评分 + 不合理项清单 + 替代方案建议

校验维度：
- **距离/时间约束**：每日总驱车时间 ≤ 上限（默认 4.5 小时净驾驶 + 0.5 小时缓冲）
- **一日游模板**：管理员预设若干"经典一日游"模板（如"乌布文化一日游"包含特定景点+餐厅清单），客户选模板时自动锁定 POI 范围；客户也可走"自由组合"模式但要过校验
- **酒店选位合理性**：当天首个景点距离酒店 ≤ 阈值（一般 1.5 小时）
- **首末日机场距离**：抵离日酒店 ≤ 1 小时车程到机场
- **AI 路线评估**：把当日完整路线（酒店→A→B→C→D→晚餐→酒店）发给 Claude，让模型用自然语言判断是否赶、节奏是否舒服、有没有重复绕路、推荐顺序优化

**数据基础**：
- `Distance` 表：景点-景点 / 景点-酒店 / 景点-机场 间的车程（分钟数 + 公里数）
- `DayTripTemplate` 表：管理员预设的一日游清单 + 适配景点/餐厅 ID 列表
- `TimeBudget` 配置：净驾驶上限、单景点游览推荐时长、堵车系数（继承自母库 PEAK_COEF：早高峰 ×1.40 / 晚高峰 ×1.55 / 节假日 ×1.65）

**返回结构**：
```json
{
  "overall_feasible": false,
  "days": [
    {"day_index": 1, "feasible": true, "drive_minutes": 180, "warnings": []},
    {"day_index": 2, "feasible": false,
     "drive_minutes": 380,
     "errors": ["Day 2 总驾驶 6h20min 超出 5h 上限",
                "京打玛尼火山到金巴兰单程 2h15min，下午 2 点出发当日无法回酒店"],
     "suggestions": [
       "把京打玛尼挪到 Day 3，Day 2 改为乌布周边（圣猴森林+德格拉朗梯田+乌布皇宫）",
       "或者保留京打玛尼但去掉金巴兰晚餐，改在 Ubud 用餐"
     ]
    }
  ]
}
```

### 6. Pricing Engine（计价引擎）
- 把 QuoteItem 列表的 IDR 成本汇总
- 按人数、车型、淡旺季、汇率换算到 CNY
- 加上利润率/赌自费让利
- 生成对外报价（仅显示 CNY 总价 + 简化拆分，不暴露成本）

### 7. Doc Generator（文档生成）
- Excel 报价单：openpyxl
- PDF 报价单：WeasyPrint（HTML → PDF，CJK 兼容方案见母库 CLAUDE.md）

## 数据流：从录入到报价

```
[Step 1] 管理员上传供应商 Excel 价格表
     │
     ▼
[Step 2] AI Parser → 提取酒店/车辆/餐厅 cost rows
     │  返回结构化 JSON（待确认）
     ▼
[Step 3] 管理员预览 → 确认入库
     │
     ▼
[Step 4] Resources 表写入
     │
     ▼
[Step 5] B 端用户在前端选行程：
     │  · 4 晚 5 天
     │  · Day 1: 抵达 + 接机 + 酒店
     │  · Day 2: 全天乌布游 + 下午茶
     │  · Day 3: 自由活动
     │  · Day 4: 自由活动
     │  · Day 5: 蓝梦岛一日游 + 送机
     ▼
[Step 6] Feasibility Engine 校验：
     │  · Day 2 京打玛尼+金巴兰超时 → 弹错并给替代方案
     │  · 用户调整后再次校验，全部 PASS
     ▼
[Step 7] Pricing Engine 汇总 IDR 成本 → CNY 报价
     │
     ▼
[Step 8] Gamble Engine 看到有 2 天自由活动，建议:
     │  "可下浮 ¥350-¥600/人，AI 信心分 78%"
     │  推荐值: ¥450
     ▼
[Step 9] 用户接受/调整 → 生成最终报价
     │
     ▼
[Step 10] 导出 Excel/PDF + 入库 Quote 表
```

## 安全与权限（v0.1 简化）

- v0.1：本地单用户运行，无认证
- v0.2：JWT 登录 + B 端旅行社账号 + 管理员/普通用户角色
- 价格表对外仅显示 RMB 总价，不暴露 IDR 成本（前端按角色渲染）

## 性能与扩展

- v0.1 SQLite 单文件：1000 个资源、500 张报价单完全够用
- AI 调用：每次 1-3 秒延迟，前端 loading 状态友好提示
- 文件上传：限制 10MB / 文件，存到 `uploads/{date}/`

## 关键技术决策记录（ADR）

| # | 决策 | 备选 | 理由 |
|---|------|------|------|
| 1 | Python + FastAPI 而非 PHP | CodeIgniter（公司在用） | AI 文档解析生态 Python 远胜 |
| 2 | SQLite 起步而非 PostgreSQL | PostgreSQL/MySQL | 零运维、单文件可备份；SQLAlchemy 切换成本极低 |
| 3 | Vue 3 CDN 而非 Vite 项目 | Vite + 单独前端项目 | 开箱即用、避免 npm；管理员/用户都能直接修改 |
| 4 | Claude（4.6）而非 GPT-4 | OpenAI / Gemini | 中文 + 多模态 + 长文档处理优势；与公司 AI 工具链一致 |
| 5 | 资源表分多张而非一张通用表 | 多态单表 | 各资源字段差异大（酒店有房型、车辆有座位数），分表查询性能与可维护性都更好 |
| 6 | 赌自费算法走 AI + 历史数据双轨 | 纯硬编码规则 | 业务策略会随季节、客群变化，硬规则维护成本高 |
