## 一、CLI 本身的下一步 (按 ROI 排)

**症状**: CLI 改 1 行可能破多个子命令,目前只能手工冒烟。
**修复**:

**为什么 P0**:CLI 是开发自用,但也是 ERP 同步/cron 任务的未来入口,坏了不能 silently fail。

### 🔴 P0 — `bws quote calc --save` 加事务保护

当前 `--save` 改了 `cost_idr_total/cost_cny_total` 但其他派生字段 (profit_cny_per_pax, gamble_cny_per_pax) 没动。
应该:要么不写,要么整体走 `routers/quotes.py` 里的完整 recalc 流程。

### 🟡 P1 — Cron 友好

CLI 现在是交互式 (有 `confirm()`),将来 cron 调度时需要:
- [ ] 所有交互点都已有 `--yes` 旁路 (✅ data restore / server stop 已加)
- [ ] 退码语义稳定:0=成功, 1=业务错误, 2=参数错误, 130=Ctrl+C — 当前散乱,需统一
- [ ] 加 `--quiet` 抑制 INFO 输出,只留 ERROR

### 🟡 P1 — `bws data import` 输出可解析

现在 import 走 subprocess 透传,日志混在一起。改成捕获 stdout 后输出 JSON summary `{"created": 123, "skipped": 4, "errors": []}`,便于 cron 把结果发到企业微信。

### 🟢 P2 — Shell 补全

argparse 不原生支持 (argcomplete 可加但要 PowerShell profile 改)。
真要做:换 typer 重写 (`backend/app/cli/__init__.py` 大改),收益是补全 + 彩色 help,代价是新增 `typer[all]` 依赖。
**评估**:工具是开发自用,补全便利度对工作流提升有限,P2 不动。

## 二、连带暴露出来的项目级问题

### 🔴 项目根没有 `pyproject.toml` → 装包混乱

本次为了 entry_points 临时加了,但 `requirements.txt` 还在 `backend/`,两边没对齐。
下次应该:
- [ ] runtime dep 从 `requirements.txt` 迁到 `pyproject.toml [project.dependencies]`
- [ ] dev dep (pytest / black / ruff) 放 `[project.optional-dependencies.dev]`
- [ ] `pip install -e ".[dev]"` 一行装齐
- [ ] CI 用 `pyproject.toml` 不再读 requirements.txt

### 🔴 `database.py init_db()` 的 ALTER TABLE 兼容层已经积累 12 条

每次新增字段都往里塞 `ALTER TABLE ... ADD COLUMN ...`,启动时全跑一遍 (失败 silently 吞)。
这是技术债 — 早就该上 Alembic。下次启动前:
- [ ] `bws db migrate` 改为真的调 Alembic
- [ ] 把现有 ALTER TABLE 列表初始化为 baseline migration
- [ ] 之后所有 schema 改动走 `alembic revision --autogenerate`

### 🟡 没有统一的"开发体验入口"

老的 `scripts/start.bat` 干硬编码事 (pip install / init_db / uvicorn) — 现在 CLI 都能做,可以:
- [ ] `bws.bat dev` = `bws db init` + `bws server start` 一条龙
- [ ] 删除 `scripts/start.bat / start.sh`,统一指向 CLI

### 🟡 测试覆盖率仍是 0

- [ ] **Playwright E2E** (上次就说过,今天还没做) — 登录 → 切 Tab → 创建报价 → 算价 → 导出
- [ ] CLI 测试 (本节 P0)
- [ ] pricing_engine 纯函数单测 (输入 Quote ORM → 输出 PricingBreakdown,业务核心,不测不放心)

## 四、思维方式 (今天验证有效)

1. **侦察先于动手** — 今天先看 main.py / config.py / pricing_engine / models,然后才写 CLI。看清依赖才不会写出导不进来的代码。
2. **零新依赖优先** — argparse 比 typer 简陋,但少一个 dep 就少一个升级面 / 安全面 / 文档面。
3. **ASCII 兜底原则** — 凡是 Windows 系统外壳能碰到的脚本 (.bat / .ps1 / git hook) 全英文,中文丢到 .py 里去。
4. **冒烟测试要走真实路径** — 不只是 `--help`,要 db stats / quote calc / server status 这种真打 DB 真发 HTTP 的,才能发现 .bat 编码问题。
