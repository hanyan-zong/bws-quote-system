## 二、按文件类型设计 Prompt

### 2.1 通用 System Prompt
```
你是一个专业的旅游成本数据抽取助手，服务于一家面向中国市场的印尼地接旅行社。
你的工作是从供应商发来的报价文件中精准抽取以下类型的资源信息：
- 酒店（hotel + room_type + cost_idr_per_night + 淡旺季）
- 景点（attraction + ticket_idr + 推荐时长）
- 餐厅（restaurant + cost_idr_per_person + 餐型）
- 车辆（vehicle + seat_count + cost_idr_per_day）
- 导游（guide + language + cost_idr_per_day）
- 下午茶/SPA/水上项目

输出规则（严格遵守）：
1. 仅返回 JSON，不要任何解释性文字
2. 货币统一用 IDR；如果原文是 USD/CNY，按 USD=15500 IDR / CNY=2300 IDR 换算并标注 source_currency
3. 不确定的字段填 null，不要瞎猜
4. 一个文件可能含多个资源，按数组返回
5. 给每条记录加 confidence (0~1)，低于 0.7 的字段在 low_confidence_fields 数组里列出
```

### 2.2 PDF 处理 Prompt（追加）
```
输入是一份 PDF 文件抽取出的文本+表格。
注意：
- 表格可能跨页，相同表头的多页要合并
- 数字可能含千位分隔符（150,000 = 150000）
- 印尼语 "kamar" = 房间，"per malam" = 每晚，"termasuk sarapan" = 含早餐
```

### 2.3 Excel 处理 Prompt（追加）
```
输入是一份 Excel 文件，按 sheet → row 顺序提供。
注意：
- 表头可能在第 2-5 行，自行识别
- 同一 sheet 可能含多个供应商表格（如不同酒店分块）
- 合并单元格已展开（每行都有完整字段）
- 公式已计算为最终值
```

### 2.4 图片处理 Prompt（追加）
```
输入是一张报价单截图或扫描件。
注意：
- 可能是手机拍照、有倾斜/反光，请尽量识别
- 数字识别错误的代价高，不确定时用 null + 标注 low_confidence
- 印尼语+英语+中文混排很常见
```

### 4.2 `app/ai/document_parser.py`
按文件类型分流到不同 handler，最后都送进 `ClaudeClient.extract()`。

```python
async def parse_document(file_path: Path) -> dict:
    suffix = file_path.suffix.lower()
    if suffix == ".pdf":
        text = await extract_pdf_text(file_path)
        content = [{"type": "text", "text": text}]
    elif suffix == ".xlsx":
        sheets = await extract_xlsx(file_path)
        content = [{"type": "text", "text": json.dumps(sheets, ensure_ascii=False)}]
    elif suffix == ".docx":
        text = await extract_docx_text(file_path)
        content = [{"type": "text", "text": text}]
    elif suffix in (".png", ".jpg", ".jpeg", ".webp"):
        image_b64 = base64.b64encode(file_path.read_bytes()).decode()
        content = [{
            "type": "image",
            "source": {"type": "base64", "media_type": f"image/{suffix.lstrip('.')}", "data": image_b64}
        }]
    else:
        raise ValueError(f"不支持的文件类型: {suffix}")
    
    return await client.extract(system=SYSTEM_PROMPT, content_blocks=content)
```

## 五、本地预解析细节

### 5.1 PDF (`pdfplumber`)
- 优先抽表格 (`page.extract_tables()`)
- 抽不到表格再回退抽文本
- 多列布局用 `page.crop()` 按 bbox 切

### 5.2 Excel (`openpyxl`)
- `data_only=True` 读计算结果
- 合并单元格展开为相同值
- 输出格式：`{"sheet_name": [[row1], [row2], ...]}`

### 5.3 DOCX (`python-docx`)
- 段落 + 表格按出现顺序拼接
- 表格行列结构保留

### 5.4 图片
- 不做本地 OCR（Claude 原生视觉能力优于 Tesseract+规则）
- 大图压缩到长边 1920px 节省 token
- 返回前在响应里附原图缩略图供前端预览

## 六、错误处理与降级

| 场景 | 处理 |
|------|------|
| API key 未配置 | 返回 mock 数据 + 提示用户配置 |
| Claude 调用超时 | 重试 1 次 → 失败则返回本地预解析的原始文本 + 友好提示 |
| 返回非 JSON | 用正则抠出 ```json...``` 块；再失败则返回原文+人工模式 |
| 字段缺失/类型错 | 标记 low_confidence + 给出可能值供前端选 |

## 七、持续优化机制

### 7.1 修正反馈写入
每次用户在前端修正 AI 结果，写入：
```
logs/ai_corrections.jsonl
```
单行 JSON：
```json
{"ts":"...","file":"...","field":"cost_idr_high","ai_value":2800000,"user_value":2900000,"reason":"原文写的是 290 万"}
```

### 7.2 周期复盘
每周一脚本统计：
- 各资源类型 Top 10 错误字段
- 哪些供应商的文件错误率高 → 找他们要标准化模板
- 输出 `logs/weekly_ai_report.md`

### 7.3 prompt 优化迭代
- 每收集 100+ 条修正 → 让 Claude 自己分析修正模式 → 输出 prompt 优化建议
- 应用 → 灰度对比新旧 prompt 准确率 → 决定是否上线

## 八、安全与合规

- 上传文件保留 30 天，超期归档到 `uploads/archive/`
- 不将客户敏感信息（联系方式/价格细节）发到 Claude — prompt 里强调"忽略个人信息字段"
- API key 通过环境变量加载，不入库不入日志
- AI 解析记录 `ai_extractions` 表保留全量，便于审计
