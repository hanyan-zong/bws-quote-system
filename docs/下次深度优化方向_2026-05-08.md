## 🔴 P0 · 立即必须做（生产前必备）

### 1. 数据库迁移工具 Alembic
**当前痛点**：v0.3 加 `gamble_strategies` 新表只能依赖 `Base.metadata.create_all()` 自动建。改字段必须 `database.py` 写 ALTER TABLE 补丁（已用过 4 次，越积越多）。生产环境每次升级都要担心改坏数据。

**做什么**：
```bash
pip install alembic
cd backend && alembic init alembic
# 编辑 alembic/env.py 让它读 Base.metadata
alembic revision --autogenerate -m "init v0.3 baseline"
alembic upgrade head
```
未来每次模型改动：`alembic revision --autogenerate -m "add X"` → `alembic upgrade head`，完全自动 + 可回滚。

**移除**：`database.py` 里那段 `additive_columns ALTER TABLE` 兼容补丁。

**预计**：1 小时。

---

### 2. 报价导出 PDF / Excel / Word
**当前痛点**：报价能在系统里看，但客户拿到的物料还是空缺。业务员现在只能截屏发给 B 端旅行社，太丢人。

**做什么**：
- `routers/quotes.py` 加 `GET /quotes/{id}/export?format=xlsx|pdf|docx`
- Excel：`openpyxl` 出"报价单 + 行程明细 + 价格汇总"3 sheet
- PDF：`weasyprint`（已在 requirements）配中文字体（已在 Dockerfile）渲染 HTML 模板
- Word：`python-docx`（已在 requirements）走 BWS 母库已经成熟的 docx 生成模式
- **关键约束**：仅显示 CNY 总价 + 简化拆分，**不暴露 IDR 成本**

**模板风格**：复用母库 `02_行程产品报价/雅加达瑟帕岛报价2026/` 已有 docx 排版。

**预计**：2 小时（Excel 30min + PDF 60min + Word 30min）。

---

### 3. 赌自费回写闭环 → 反哺策略库
**当前痛点**：策略库是经验值，没"自学习"。`gamble_history` 表存在但没有"团结束财务回写实际自费收益"的入口。3 个月后策略好不好用，没数据说话。

**做什么**：
- 报价历史 Tab 每条加 "✅ 团结束 录入自费数据" 按钮
- 表单：`实际自费总收入 ¥` + `实际利润 ¥` + `won/lost` + 备注
- POST `/quotes/{id}/feedback` → 写 `gamble_history.optional_tours_revenue_cny / profit_actual_cny / won_or_lost`
- Settings 加 "📈 策略胜率统计" 卡片：每条 GambleStrategy 显示 `命中次数 / 中标率 / 平均实际利润`

**进阶（v0.4）**：当某条策略胜率 < 60% 持续 10 单 → 系统自动 toast "建议调整 X 策略"。

**预计**：1.5 小时。

## 🟡 P1 · 下次迭代（提升业务）

### 4. 多用户系统 + JWT
**当前**：单租户口令门（admin/123456）。所有 B 端旅行社共用一份资源库 + 报价历史，看得到对方数据。

**做什么**：
- 新表 `users`（id / username / password_hash / role: admin/agent/viewer / agency_id）
- 新表 `agencies`（id / name / contact / commission_rate）
- 老 cookie HMAC → JWT (`PyJWT`)，header 带 token
- 报价、AI 解析记录、上传文件都按 agency_id 隔离
- Admin 看全部 / agent 看本 agency / viewer 只读

**预计**：3-4 小时。

---

### 5. 真距离矩阵 → Google Distance Matrix API
**当前**：`distances` 表是 seed 硬编码，新景点之间没数据时走默认 30min（可能完全偏离实际）。

**做什么**：
- 新增设置项 `GOOGLE_MAPS_API_KEY` 走环境变量
- `utils/distance_filler.py`：批量调 Distance Matrix API → 写 `distances` 表
- Admin 面板加按钮"🌍 一键回填距离矩阵"
- 缓存：30 天 TTL，超期自动重拉
- 节流：每秒 ≤ 10 请求避免封 key

**预计**：2 小时（含 API key 申请走流程）。

**替代方案**：自建 OSRM 服务（OSM 数据，免费但要服务器）。

---

### 6. AI 行程评估深度化（输出可应用 patch）
**当前**：AI 评估返回 `{score, issues, improved_route}` — 都是文字，用户看完还要自己改。

**做什么**：让 AI 返回结构化 patch：
```json
{
  "score": 6,
  "issues": ["Day 2 总驾驶 6h 超时"],
  "patches": [
    {"type": "remove_attraction", "day_index": 2, "attraction_id": 18, "reason": "京打玛尼太远"},
    {"type": "swap_hotel", "day_index": 3, "hotel_id_from": 5, "hotel_id_to": 12, "reason": "换近点的酒店"}
  ]
}
```
前端"💡 一键应用"按钮自动修改 quote 重新计算。

**预计**：1.5 小时。

---

### 7. Strategy 与 OptionalTour 联动展示
**当前 v0.3 缺失**：命中 fixed 策略后直接返回金额，前端不再显示"配套自费推荐"。但实际上业务员需要看到"我让了 ¥450 是希望客人会买这 N 个自费"的清单。

**做什么**：在 GambleResult 即使命中 strategy 也带上 `configured_optional_tours` + `excluded_optional_tours`。前端"赌自费推荐"卡片照常展示。

**预计**：30 分钟（小修复但 UX 重要）。

## 🟢 P2 · 平台化（规模复制）

### 8. ERP 订单同步接口
**痛点**：`Quote.status='accepted'` 后人工抄到母库 ERP（07_ERP系统），易丢易错。

**做什么**：
- 后端加 webhook：quote 状态变 accepted → 推 POST 到 ERP `/api/orders/import`
- ERP 端建对接 API（PHP/CodeIgniter）
- 失败重试 3 次 + 失败事件入 `webhook_failures` 表
- Admin 面板"🔄 ERP 同步状态"

**预计**：2-3 小时（含 ERP 端开发）。

---

### 9. 一日游模板自动归纳（AI）
**当前**：模板靠管理员维护。123 个景点能组合出几百条"经典路线"，靠人工想不到这么多。

**做什么**：
- 跑批：取过去成单 100+ 行程的 Day 数据
- 按景点组合 + 频次统计 + 区域聚合 → 生成 candidate templates
- Claude 命名 + 描述
- 管理员审核后批量入库（标 `created_by="ai_v1"`）

**预计**：3 小时。

---

### 10. 微信小程序版本（B 端业务员手机端选行程）
**痛点**：业务员经常出差，电脑不在身边但客户在群里问报价。

**做什么**：
- Vue 3 → uni-app 重写 QuoteBuilder
- 后端 API 不动（已是 RESTful + JWT）
- 微信公众号绑定登录

**预计**：1 周（重活）。

## 🔵 P3 · 差异化竞争（半年内）

### 11. AI 销售助手（对话式接询单）
> 业务员说："4 月去巴厘 6 天蜜月" → AI 自动选模板 + 酒店 + 车 + 景点 → 30 秒出报价。

落到"5 分钟出报价"目标的最后一公里。

**预计**：1 周（含 prompt 工程 + 多轮对话状态机）。

---

### 12. 反向定价（客户预算驱动）
> 客户："8000/人 7 天" → 系统反推可行配置（4-5 套备选）。

匹配 B 端真实询单流程（先有预算后选行程，而不是先选行程后报价）。

**预计**：1 周。

---

### 13. ML 替代赌自费经验值
3 个月后收集 30+ 单 `gamble_history`：
- 用 sklearn / XGBoost 训预测模型
- 特征：行程结构 + 客户类型 + 季节 + 含资源 → 期望自费毛利
- 替代当前的 GambleStrategy 经验值表（或并存做对照实验）

**预计**：2-3 天（含数据预处理 + 调参）。

## 监控与可观测性（缺失项）

当前完全没有：
- **错误聚合**：Sentry / Glitchtip
- **API 性能**：调用次数 / P95 / 慢端点
- **AI 调用统计**：token 消耗 / 成功率 / 平均延迟
- **业务 KPI**：日生成报价数 / 成单率 / 平均售价

**建议**：v0.4 引入轻量 Prometheus + Grafana 或托管 Sentry，先解决错误聚合（最高 ROI）。
