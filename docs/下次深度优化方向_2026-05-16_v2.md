## 二、🔴 P0 settings 重构 — 消除 sys.modules reload hack

### 现状

`backend/tests/web/conftest.py` 顶部用了**侵入式** sys.modules reload:

```python
os.environ["BWS_DATABASE_URL"] = f"sqlite:///{_TMP_DB_PATH}"
for _mod in list(sys.modules):
    if _mod == "app" or _mod.startswith("app."):
        del sys.modules[_mod]
from app.database import engine, init_db
```

这是因为 `app.config.Settings` 用 **class-level os.getenv**,import 时锁定。

### 问题

- 任何"在主进程 import 后想换 DB"的场景都得用这个 hack
- 测试可读性差,新增 web 测试文件得继承同样的 reload 步骤(否则单跑/混跑行为不一致)
- 如果未来 CI 跑测试时有第三方 plugin 也 import app,可能链式踩坑

### 修复方案(标准 FastAPI 做法)

**方案 A: pydantic-settings + lazy property**(轻量,1h)
```python
# app/config.py
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    database_url: str = f"sqlite:///{DATA_DIR / 'bws_quote.db'}"
    # 自动读 BWS_DATABASE_URL env (model_config.env_prefix="BWS_")
    model_config = {"env_prefix": "BWS_"}

@lru_cache
def get_settings() -> Settings:
    return Settings()
```

但 `engine = create_engine(settings.database_url)` 仍是 module-level,改 settings 不变 engine 的 URL。需要进一步:

**方案 B: engine 也 lazy + FastAPI dependency**(彻底,2-3h)
```python
# app/database.py
@lru_cache
def get_engine() -> Engine:
    return create_engine(get_settings().database_url, ...)

def get_db() -> Generator[Session, None, None]:
    SessionLocal = sessionmaker(bind=get_engine(), ...)
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

测试用 FastAPI 的 `app.dependency_overrides[get_db] = lambda: tmp_session()` 覆盖,**不再需要 sys.modules reload**。

### 工期 & 风险

- 改 config.py + database.py + 所有调 `engine` / `SessionLocal` 的地方(可能 10+ 个文件)
- 风险点:某些 module-level 用 `from .database import engine` 的代码需要改成 `from .database import get_engine`,**懒求值时机**可能影响 startup 顺序
- 测试需要重写 conftest.py(去掉 reload hack,改用 dependency_overrides)
- **价值**:从此测试隔离干净,不再有"为什么混跑 fail 单跑 PASS"的奇怪 bug

## 三、🟡 P1 datetime.utcnow() deprecation 全局替换

### 现状

测试运行时 **52 个 DeprecationWarning**:

```
DeprecationWarning: datetime.datetime.utcnow() is deprecated and scheduled for removal in a future version.
Use timezone-aware objects to represent datetimes in UTC: datetime.datetime.now(datetime.UTC).
```

涉及文件(grep 找过):
- `backend/app/routers/auth.py`(last_login_at, locked_until)
- `backend/app/routers/quotes.py`(_generate_quote_no)
- `backend/app/utils/feature_permissions.py`
- `backend/app/models/*.py`(`default=datetime.utcnow` 在 SQLAlchemy 字段定义里)
- 可能还有其他

### 修复

全局替换 `datetime.utcnow()` → `datetime.now(UTC)`,加 `from datetime import UTC`(Python 3.11+ 才有 UTC sentinel,3.12 OK)。

注意:SQLAlchemy `Column(DateTime, default=datetime.utcnow)` 改成 `default=lambda: datetime.now(UTC)`(因为 `datetime.now(UTC)` 是 call,SQLAlchemy 期望 callable)。

### 工期 & 收益

- 工期 30-60 分钟(全 grep 替换 + 测试)
- 收益:Python 3.14 以后 utcnow() 会真的被移除,提前修避免硬故障;测试输出干净 52 warnings

## 四、🟡 P1 pyproject 加 `[project.optional-dependencies] test`

### 现状

`pyproject.toml` 没有 test deps 段。pytest / alembic 都是手动 `pip install`,新机器装项目时不知道要装哪些测试依赖。

### 修复

```toml
[project.optional-dependencies]
test = ["pytest>=9.0", "httpx>=0.27"]
# 如果未来加 pytest-xdist 也写这里

[project.optional-dependencies.dev]
dev = ["pytest>=9.0", "httpx>=0.27", "ruff", "mypy"]
```

之后跑 `pip install -e .[test]` 一次装全。

### 工期

5 分钟。

## 六、🟢 P2 pytest-xdist 加速混跑

### 现状

混跑 28s 中,CLI subprocess 模式占 24s(每个 subprocess 起 SQLAlchemy + bootstrap admin)。web 测试 0.71s。

### 修复

`pip install pytest-xdist` + `pytest -n auto`,把 30 个 CLI subprocess 测试并行跑(每 subprocess 独立 DB fixture,无共享状态)。

但 web 测试**不能** parallel(共享 session-scoped tmp DB),要用 `-p no:xdist` 在 web 段排除,或者 `pytest -n auto backend/tests/test_cli_*.py && pytest backend/tests/web/`。

### 工期 & 收益

- 工期 30 分钟(装 + conftest 配 + 验证)
- 收益:CLI 30 用例从 24s → 估计 6s(4 核并行),全套件可能 10s 内完成
