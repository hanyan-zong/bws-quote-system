## 五、技术选型与避坑（精华版）

### 选型
| 层 | 选择 | 关键理由 |
|----|------|---------|
| 后端 | Python FastAPI | AI 文档解析生态最近 |
| ORM | SQLAlchemy 2.x | 行业标准 + Pydantic 互通 |
| DB | SQLite → PG | 单文件起步，未来无缝迁移 |
| 前端 | Vue 3 CDN + Element Plus | 零构建，单 HTML 即用 |
| AI | Claude Sonnet 4.6 | 中文 + 多模态 + 长文档优势 |
| 文档解析 | pdfplumber / openpyxl / python-docx + Claude 多模态 | 先抽结构后送 AI |

### 避坑（含母库继承 + 今日新踩）
1. **SQLite on FUSE 挂载盘**：`disk I/O error`. 解决：`BWS_DATABASE_URL=sqlite:////tmp/...` 或在 start.sh 自动检测
2. **Pydantic v2 + Python 3.10 字段名遮蔽**：`date: date | None` 报 `unsupported operand`. 解决：`from datetime import date as _date` 用 `Optional[_date]`
3. **v0.2 字段迁移**：`Base.metadata.create_all` 不加列, 必须删旧 db 重建
4. **partial-meal 判断 elif 没赋值**：今日 smoke test 暴露的逻辑 bug, 已修
5. **quote_no 同秒重复**：精度只到秒导致 `UNIQUE constraint`. 已加 random 3 位后缀
6. **gamble result 序列化漏字段**：v0.2 新增 skip_rule / excluded 字段忘了在 quotes/gamble router 里返回. 已修
7. **GambleConfig.first_time_agency_factor 仅减赌额未减为 0**: 这条护栏配合 NoGambleRule "首次合作 + 小团" 双层兜底
8. **docx-js 中文引号必须 「」**：母库继承
9. **SVG `&` 必须转义**：母库继承
10. **CJK 字体仅 DroidSansFallbackFull**：母库继承

## 六、下次深度思考的优化方向

按业务价值与技术债务双维度排序。

### 🔴 P0 · 立即必须做（影响生产可用）

#### 1. 数据库迁移 (Alembic)
- 当前 v0.2 改字段需要删 db 重建，生产不可接受
- 引入 Alembic：`alembic init alembic` → 每次模型改动生成迁移脚本
- 历史报价数据保留，结构变更平滑

#### 2. 多用户 + 权限
- v0.1/0.2 是单用户本地版
- 引入 JWT + 角色：admin（全部）/ agent（B 端用户，仅报价）/ viewer（只读）
- B 端旅行社账号按 agency 隔离数据

#### 3. AI key 安全管理
- 当前从环境变量读，多用户场景需要加密入库
- 用 Fernet 对称加密存到 settings 表，运行期解密

#### 4. 文档导出（Excel + PDF + Word）
- 已设计但未实现 export endpoint
- openpyxl 出 Excel；WeasyPrint 出 PDF（参考母库 weasyprint 经验）
- 客户版只显示 CNY，不暴露成本拆分

### 🟡 P1 · 下次迭代（提升业务价值）

#### 5. 真距离矩阵接入
- 当前 distance 是 seed 硬编码
- 接 Google Distance Matrix API 自动批量回填
- 或者 OSM + OSRM 自建路由服务（免费）

#### 6. 赌自费历史 → ML 预测
- 收集 30+ 单 gamble_history 后, 用 sklearn 或 XGBoost 训预测模型
- 替代当前的经验值表
- 输入: 行程结构 → 输出: 赌额预测 + 置信区间

#### 7. 行程合理性 AI 评估深度
- 当前 AI 评估仅返回 score + issues，未深入到"重排建议"
- 应让 AI 直接返回 patch（具体改哪个字段 → 改成什么），用户一键应用
- 参考文档 `07_行程合理性校验.md` 第 8 节

#### 8. 一键应用建议
- 校验失败时给的 suggestions 现在只是文字
- 应做成 patch JSON，前端 click "应用"按钮直接修改 quote 重新计算
- 比"用户读完再手动改"快 10 倍

#### 9. 不赌规则增加 OR/AND 嵌套
- 当前只支持 AND（一条规则的 conditions 全部命中）
- 实际场景：(蜜月 OR 婚礼) AND (短行程 OR 全含餐)
- DSL 升级到树形结构

#### 10. AI Prompt 持续优化机制落地
- 已有 ai_corrections.jsonl 日志位
- 每周自动跑：分析 Top 错误字段 → Claude 反思 prompt 漏洞 → 灰度新 prompt
- 目标：3 个月内 AI 解析准确率 95% → 99%

### 🟢 P2 · 平台化（规模复制）

#### 11. 多目的地多日联动校验
- 当前校验只到单日
- "雅加达 → 巴厘 → 龙目" 三段联动需要：航班对接、时区转换、行李托运
- 设计 `LegContext` 跨目的地的连接信息

#### 12. 微信小程序版本
- 客户在小程序选行程 → B 端业务员后台审批 → 报价回写
- 前端 Vue 3 → uni-app 重写

#### 13. ERP 订单同步
- 与公司现有 07_ERP系统 打通
- quote.status='accepted' 时自动推订单到 ERP

#### 14. 供应商门户
- 酒店/车队 自助维护成本 + 房态 + 调价
- 减少人工录入；Q1 可减少 50% 录入工作量

#### 15. 一日游模板自动归纳
- 让 AI 分析过去 100 单成功行程
- 自动归纳 5-10 个新模板补充到 day_trip_templates
- 模板有 created_by="ai" 标记

### 🔵 P3 · 进阶能力（差异化）

#### 16. 实时 OTA 房价对接
- 接 Hotels Combined / RateHawk 等批发商接口
- 自动比价；避免供应商单边涨价吃利润

#### 17. 行程图片化输出
- 自动生成行程地图（参考母库巴厘岛地图制作经验）
- 客户版报价单嵌入地图 + POI 缩略图

#### 18. AI 销售助手
- 对话式接询单："我想 4 月去巴厘 6 天蜜月" → AI 自动选模板 + 酒店 + 车 → 出报价
- 真正落到"5 分钟出报价"的目标

#### 19. 反向定价（客户预算驱动）
- 客户给预算（"8000/人 7 天"）→ 系统反推可行配置
- 比当前的"先选行程后报价"更符合 B 端真实询单流程

#### 20. 与母库 bws-toolkit 双向打通
- 当前 bws-toolkit 是命令行 docx/excel 生成
- 升级为：本系统 quote → 一键调 bws-toolkit 生成对外文档
- 复用母库的 JSON 产品规范
