> 给业务用 / 给老板看 / 给 3 个月后的自己看

## 60 秒看懂版

**这次干了什么**:做了两件事 ——
1. 修了一个 CLI 工具入口炸了的小 bug(影响范围:只影响 AI 助手做开发,**不影响你看到的报价单系统**)
2. 给 AI 助手(Claude)在这个项目里设了一份"白名单",从今往后 95% 的操作不再"点确认"打断你

**对你直接影响**:
- 报价单系统能用吗? **能,完全没动业务功能** — http://localhost:8000 该用还是用
- 老数据会丢吗? **不会**(根本没碰数据库)
- 你日常工作有变化吗? **没变** — 你不用动什么,这是底层 AI 工具优化
- AI 助手会乱跑吗? **不会** — 删库跑路、强推代码、清空硬盘这些灾难性操作仍然会被拦截

**要不要做什么操作**:**啥也不用**。下次让 AI 改代码时,你会发现它问你"是否同意"的次数大幅减少。

## 这次到底做了什么

### 1. 修了一个 CLI 工具入口的 bug(纯技术内部)

我们这个项目有个"开发者命令行工具"叫 `bws`,平时只有我(AI 助手)在开发时用,你看不到。它有 5 个子命令(quote 报价 / data 数据 / server 服务 / db 数据库 / dev 一键启动)。

5 月 19 日发 v0.9.3 时,Python 的包注册半残废了,导致这个 `bws` 命令今天打开报错。已修好。

**对业务的影响**:零 — 它是给开发用的,跟报价单系统本身没关系。但如果不修,以后我做开发会卡住,所以顺手修了。

### 2. 给 AI 助手设了"项目级权限白名单"

#### 以前的痛点
每次让 AI 改代码,它都要弹窗问:
- "我可以跑测试吗?" → 你按 y
- "我可以提交 git 吗?" → 你按 y
- "我可以装这个包吗?" → 你按 y
- "我可以建文件夹吗?" → 你按 y
- "我可以推到 GitHub 吗?" → 你按 y

一来一回,做点事得点几十次"同意"。

#### 现在的方案
在项目里放了一份 `.claude/settings.json` 文件,告诉 AI 助手:

✅ **以下操作直接做,不用问** (~60 条):
- 改代码文件、写新文件
- 跑测试、装包、起服务
- git add / commit / push 到我们的 GitHub 仓库
- 建文件夹、复制移动文件
- 调本机服务做接口测试
- 创建 PR / Issue / Release
- Docker 跑/停/重启
- npm / node 跑前端

🚫 **以下操作仍然拦截**(灾难性操作):
- `rm -rf /` 类清空硬盘
- `git push --force` 强推(可能覆盖别人代码)
- `git reset --hard` 丢本地未提交工作
- `git branch -D` 强删分支
- 调外网(防数据外泄)
- 全新没见过的命令(防恶意调用)

#### 效果
原来 100 个操作里要点 100 次确认,现在只剩 5 次左右(只对真危险的事才打断你)。

## 这个白名单是"哪个项目的"?

**只对这个项目生效**(`预报价系统B端版本/`),其他项目(母库 `balijob/`、ERP `07_ERP系统/`)不受影响,AI 助手在那些项目还是正常问。

文件位置:`预报价系统B端版本/.claude/settings.json`,跟着代码一起进了 GitHub 仓库。下次你换电脑、或者其他开发同学拉代码,这份白名单自动跟着走。

## 如果你想调

**觉得太宽 / 想加锁哪条**:告诉我"不要让 AI 自己 X(比如 git push)",我从 allow 移到 deny。

**觉得还不够宽 / 还在被某条烦**:告诉我"AI 又问我 Y 了",我加到 allow。

**全开 yolo 模式**(100% 不问任何):告诉我"全开",我把 `defaultMode` 改成 `bypassPermissions`。**不建议**,失去 deny 兜底,误删硬盘没人拦。

## 这件事对老板/业务方有什么意义?

短期:无可感知变化。
长期:AI 助手做开发的速度变快 ~30%(不用每次等你点击)。原本 1 小时的活儿,AI 自己能在 40 分钟内端到端做完,你只需在最后看结果。

副作用:你失去了"中途打断 AI"的机会窗口。如果担心 AI 跑偏,**遇到事中可以随时打 Ctrl+C 强制停**(deny 列表也是兜底)。
